# /sdd:second-opinion

Независимая верификация результата `/sdd:dry-run` через **второго агента с чистым контекстом**. Защита от *self-confirmation bias*: одна и та же модель в одной и той же сессии может незаметно убедить себя, что её Dry Run верен.

Заимствовано из `tylerburleigh/claude-sdd-toolkit` (`sdd-plan-review` и `sdd-fidelity-review` — multi-model consultation), но адаптировано для one-model setup через sub-agent isolation.

## Когда применять

✅ **Использовать перед:**

- Major bump L0 — критично иметь второе мнение
- Решением, изменяющим инвариант с `precedence: critical`
- Релизом с большим количеством новых ART/ENT/INV/SCN
- Любой задачей, где цена ошибки выше цены 5 минут второго прогона

❌ **Не использовать для:**

- Тривиальных фиксов (используйте `/sdd:bug`)
- Задач с `verdict: BLOCK` после первого Dry Run — сначала разберитесь с блокером
- Routine обновлений с `priority: MAY`

## Как это работает

Команда **не запускает другую модель** (это требует отдельной инфраструктуры). Вместо этого она использует **Task tool** Claude Code для запуска суб-агента в **чистом контекстном окне**, который независимо повторяет Dry Run и сравнивает выводы.

Защита работает за счёт **изоляции контекста**: у суб-агента нет reasoning trail основной сессии, нет «я уже подумал об этом», нет привычки к выбранным решениям. Контекст забывает всё, кроме исходных спек и задачи.

## Шаги

1. **Подтвердить наличие исходного Dry Run.** Если в текущей сессии не было `/sdd:dry-run` — запустить его сначала. Не имеет смысла второе мнение без первого.

2. **Сформировать запрос для суб-агента.** Включает:
   - **Задача** — дословно как была сформулирована изначально
   - **Бандл спек** — те же L0/L1/L2 + patterns + glossary, что подавались первому Dry Run
   - **Инструкция:** «Выполни полный `/sdd:dry-run` независимо. Не смотри на результаты предыдущих прогонов. Выдай свой Verdict.»
   - **Запрет на чтение текущей сессии** — суб-агент не должен искать предыдущий ответ

3. **Запустить через Task tool.**

   ```
   Use the general-purpose subagent_type. Pass the task verbatim plus the spec
   bundle. Do not pass any reasoning or notes from the current session.
   ```

4. **Получить результат суб-агента.** Это будет полноформатный Dry Run-отчёт.

5. **Сравнить два прогона:**

   | Поле | Первый | Второй | Сходится? |
   |---|---|---|---|
   | `Touched` | <ID list> | <ID list> | да/нет |
   | `Contradictions` | <count> | <count> | да/нет |
   | `Missing specs` | <count> | <count> | да/нет |
   | `Ambiguous terms` | <count> | <count> | да/нет |
   | `Verdict` | PROCEED/BLOCK | PROCEED/BLOCK | да/нет |

6. **Принять решение по конвергенции:**

   - **Полная конвергенция** (все поля идентичны) → Verdict подтверждён, можно действовать
   - **Конвергенция Verdict, расхождение по деталям** → расхождения вынести в эскалацию: оба прогона согласны с выводом, но расходятся в обосновании — это сигнал о неоднозначности спек
   - **Расхождение Verdict** (один PROCEED, другой BLOCK) → **STOP**. Это сильный сигнал, что Dry Run хрупок. Не действовать. Эскалировать к человеку.

7. **Записать результат** в `specs/dry-run-log.md` как обычный Dry Run, но с дополнительным полем:

   ```yaml
   ---
   run: 2026-04-11T13:00:00Z
   task_summary: "..."
   touched: [...]
   verdict: PROCEED
   second_opinion:
     verdict: PROCEED
     converged: true
     differences: none
   ---
   ```

## Файлы

- **Читает:** `memory/patterns.md`, `memory/patterns/**`, `specs/**`
- **Пишет:** `specs/dry-run-log.md` (append, с полем `second_opinion`)
- **Запускает:** Task tool → general-purpose sub-agent в свежем контексте

## Правила

- **Изоляция обязательна.** Суб-агент не получает текущий reasoning trail, только исходные спеки и задачу.
- **Запрос дословный.** Не переформулировать задачу для суб-агента — это введёт bias.
- **Verdict-конфликт = STOP.** Никаких компромиссов. Если два прогона разошлись по Verdict — значит спеки или контекст недостаточно детерминированы для генерации кода.
- **Не злоупотреблять.** Second opinion стоит токенов и времени. Применять только перед критическими решениями.
- **Не путать с peer review.** Это не human review кода — это independent verification модели в свежем контексте.

## Связь с phase restart

Second opinion — это **частный случай** phase restart (см. [`03-context-injection.md` §9](../03-context-injection.md#9-context-rot-и-phase-restart)). Разница:

- **Phase restart** делается между фазами одного процесса (Dry Run → Plan → Code)
- **Second opinion** делается **внутри одной фазы** (Dry Run → Dry Run-2) для верификации

Оба приёма работают по одному принципу: **изоляция контекста для борьбы с context rot и self-confirmation bias**.
